# 小说头脑风暴

## 概览
通过结构化追问，把一个模糊的故事念头推成更可写的形状。这个 Skill 不替代作者的品味和判断，它只是帮助暴露约束、分支和后果。

## 何时使用
- 作者手里只有题材、场景、人物关系或一条模糊冲突线时
- 大纲太空，暂时还不能直接扩写时
- 作者在正式开写前还需要更多约束条件时
- 如果章节提纲已经足够清楚，就不必再用它

## 输入材料
- 当前想法，即使很不完整也可以
- 作者已经明确的限制条件
- 可选参考材料、主题或目标类型

## 工作流程
1. 先复述作者已经确定的内容，区分事实和猜测
2. 继续追问主矛盾、动机、限制、风险和结尾方向
3. 只有在当前约束清楚后，再给出 2 到 3 条可能分支
4. 把每条分支继续往下推一层，说明它会改变什么、增加什么代价、带来什么后续回报
5. 最后压成一份更清楚的大纲草稿或决策说明

## 输出结果
- 一个更清楚的故事前提
- 一组被明确下来的限制条件
- 一份简短大纲草稿或下一步问题清单

## 约束条件
- 不假装知道作者还没决定的内容
- 不为了显得有创意而强塞花哨转折
- 给出的分支要可比较，方便作者真正取舍
- 最后要落到可继续写的材料，而不是无止境发散

## 可定制项
- 增加类型特有的问题，例如推理的线索公平性、幻想的能力规则
- 要求 Agent 少做横向发散，多做纵向追问
- 要求 Agent 在产出一份可用大纲后就停止，而不是继续铺很多方案
