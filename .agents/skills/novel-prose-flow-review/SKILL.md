# 文笔与流畅性复审

## Overview

这是独立审稿 skill，不负责生成正文。它只检查文本读起来顺不顺、密不密、虚不虚，以及有没有偏离既定文风锚点，而不是替作者改写成另一种风格。

## Trigger Keywords

- 查文笔
- 看顺不顺
- 会不会太平
- 会不会太浮
- 检查节奏

## State Contract: Inputs

必读字段：

- `style_anchors`
- 当前草稿版本
- 本轮检查范围

可选字段：

- `target_word_range`
- `scene_summary`

忽略字段：

- 与文笔无关的设定争议
- 只表达个人喜恶、没有证据的评价

## What to Check

重点检查：

- 句子是否生硬、拧巴、信息堆积
- 段落是否空转、重复、解释腔过重
- 场景是否在流动，还是停在说明里
- 对话是否像同一个人说话
- 节奏是否塌陷，推进是否被废句拖住
- 细节是否干瘪，或者反过来堆得太满
- **是否写成流水账（平淡）**：句子只按顺序交代"谁做了什么、谁说了什么"，但关系、信息或情绪没有升级。必须提供具体证据句
- **是否过度拔高（浮夸）**：修辞、形容词、判断语气持续高位，已经压过场景本身能承载的力度。必须提供具体证据句
- 段尾和段间是否有下压：上一句留下的问题，下一句有没有接住，还是不断平推
- 是否偏离 `style_anchors`：句式、句长、内心独白密度、禁用词是否与项目既定文风明显不一致

## Output Format

每条问题都要包含：

- 严重级别：关键 / 重要 / 次要
- 问题描述
- 证据句
- 为什么影响阅读流畅性或偏离文风
- 最小修订建议

整份复审结果还应额外给出：

- 复审结论：`passed` / `needs_revision` / `pending_independent_review`
- 本轮复审范围：检查了哪一场、哪一段或哪一版草稿
- 若判定通过：明确写出本轮在节奏、句子、段间推进和文风锚点上成立的关键点

## Few-Shot

通过样例：

```text
结论：passed
关键点：
- 句子长短有变化，但整体保持克制
- 内心独白密度维持在低到中，没有突然变成大段自我解释
- 段尾持续留下小压力，下一段能接住
```

需修订样例：

```text
严重级别：重要
问题描述：段落写成顺序交代，关系压力没有上升
证据句："她坐下，端起杯子，喝了一口水。周临看着她，没有说话。她也没有说话。过了一会儿，他问她要不要继续。"
为什么影响阅读流畅性：动作和对白都在发生，但信息、关系和情绪没有任何升级，形成流水账
最小修订建议：保留动作骨架，在其中至少补出一次试探、回避或压迫，让段落承担一个明确推进动作
结论：needs_revision
```

## Red Flags - STOP and Start Over

- "我写的草稿我最懂，不用交接给别人复审"
- "文笔不好，我直接替他全部重写一遍吧"
- "我觉得太平淡了，但给不出具体证据句"
- "这段情感很强烈，所以用词浮夸一点没关系"

**如果出现以上情况：立即停止，不要直接重写正文，必须给出具体证据句和最小修订建议。**

## Independent Review Rule

- 这是**独立审稿 skill**。如果你就是当前草稿的起草者，立刻停止，不做文笔复审判定。
- 自查场景下，结论字段只能写 `pending_independent_review`，绝不允许写 `passed`。
- 若当前环境没有子智能体或第二个独立智能体，不要只输出"待独立复审"清单，必须输出可粘贴到新会话的复审启动 prompt。

降级模板：

```text
请使用 `novel-prose-flow-review` 独立复审这份草稿。

草稿版本：<draft_version>
检查范围：<scene_or_paragraph_range>
style_anchors：<style_anchors>
草稿正文：<draft_text>

要求：
- 你不是起草者
- 只检查句子、节奏、重复、说明密度和 style_anchors 偏移
- 结论只允许是 passed / needs_revision / pending_independent_review
- 若发现问题，给出严重级别、证据句、原因、最小修订建议
```

## Rules

- 不要把个人口味伪装成硬错误。
- 优先指出真正拖累阅读的地方，不要做礼节性挑刺。
- 没有证据，不要泛泛说"文笔不好"。
- 除非用户明确要求，不要直接整段重写。
- "克制"不等于"平淡"。只有当文本没有形成有效推进，只剩顺序交代时，才判为平淡或流水账。
- "强烈"不等于"浮夸"。只有当措辞力度明显脱离人物当下动作、感知和场景压力时，才判为浮夸。
- 指出"平淡"或"浮夸"时，必须点出触发它的证据句和具体症状，不能只贴标签。
- 如果问题集中在少数句段，只指出最小修订范围，不要把复审写成全面改稿命令。
