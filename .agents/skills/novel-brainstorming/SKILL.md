# 小说头脑风暴

## Overview

把发散的小说念头压成可写材料。重点不是"替作者想花活"，而是把冲突、限制、代价和方向压清楚。

## Trigger Keywords

- 脑暴
- 先想想
- 只有一个点子
- 不知道怎么起
- 这一章该写什么

## When to Use

适用于：

- 只有一句设定或一个关系钩子
- 有氛围，但没有主矛盾
- 角色很多，但不知道谁推动剧情
- 想写一章，却不知道这一章真正要完成什么

不适用于：

- 已有清楚场景计划，直接进入扩写更合适
- 只是润色已有正文

## Process

- 一次只追一个关键问题，不要同时抛一串问题。
- 先读用户已经给出的材料，再判断真正不清楚的点，不要机械追问。
- 如果大方向已经明显，只缺一两个关键空位，先提出一个**工作假设**，再请用户确认或改写。
- 主动确认的重点是：这章到底要推进什么、关系底色是什么、当前不能越过的边界是什么。
- 先固定：谁想要什么、谁阻止什么、失败代价是什么。
- 再固定：故事走向靠什么推进，是误会、秘密、资源争夺、身份压力，还是价值冲突。
- 最后压成可交付材料，例如：
  - 一句话前提
  - 主矛盾
  - 角色关系
  - 章节目标
  - 禁区与边界

## Output

输出只保留对后续写作真正有用的材料：

- 可写的故事前提
- 主要角色关系
- 当前阶段最值得推进的冲突
- 还未决定但必须明确的问题清单

建议显式分成两层，避免下游把猜测当事实：

- 已确认：用户已经明确拍板的前提、关系、边界
- 工作假设：为了继续推进而临时成立、仍待用户确认的判断

如果准备交给 `novel-constraints-organizer` 或 `novel-chapter-planning`，至少给出这几个字段：

- 故事前提
- 主矛盾
- 谁想要什么、谁在阻止什么
- 失败代价
- 本章或下一场最值得推进的方向
- 禁区与边界
- 待确认问题

## Red Flags - STOP and Start Over

- "我给作者写个完整的10章大纲吧（太早收敛）"
- "既然这个点不确定，我连问5个问题让作者选（信息过载）"
- "为了冲突而冲突，安排女配突然黑化"
- "作者的想法太平淡，我直接抛弃他的方向重新构思"

**如果出现以上情况：立即停止，只聚焦当前最需要解决的核心矛盾。**

## Rules

- 不要为了显得有创意，强塞反转。
- 不要把没定下来的东西说成既定事实。
- 用户没说清时，不要直接替他定死；先给出你理解中的工作假设，并明确哪些点仍待确认。
- 如果信息已经足够收敛到可写前提，就不要为了"完整"继续追问。
- 一旦已经能写，就停止发散，转入规划或扩写。
- 如果后续要进入长期状态层，先经过 `novel-constraints-organizer` 固化，不要让 brainstorm 结果直接充当长期设定。
