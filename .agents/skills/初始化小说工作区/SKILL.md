# 初始化小说工作区

## 概览
在正式写作前先建立最小可用的小说工作区。重点不是生成故事内容，而是准备后续扩写、摘要和检查都会反复读取的稳定文件。

## 何时使用
- 新小说项目从空目录开始时
- 资料、设定、草稿分散，需要先整理稳定位置时
- 如果项目已经有清楚目录结构，就不必重复执行

## 输入材料
- 项目标题
- 类型或题材
- 一句话核心设定
- 已知主要人物
- 可选目录，例如 `线索`、`时间线`、`素材`

## 工作流程
1. 创建作者希望长期保留的基础目录
2. 创建 `总纲.md`、`人物设定.md`、`世界观.md` 等核心文件
3. 写入空白或最小骨架标题
4. 对未知内容保持留白，不主动虚构

## 输出结果
- 一套可复用的工作区目录
- 一批后续可持续读取和更新的 Markdown 文件

## 约束条件
- 不发明重要剧情转折
- 不扩展作者没有提供的世界规则
- 优先使用 Markdown，因为很多 Agent 对 Word 支持较差

## 可定制项
- 增加 `线索/`、`时间线/`、`素材/` 等目录
- 按作者自己的习惯重命名目录
