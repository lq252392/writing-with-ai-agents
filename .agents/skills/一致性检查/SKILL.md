# 一致性检查

## 概览
对照工作区检查是否发生漂移。这个 Skill 负责检查连续性和约束冲突，不替代作者的文学判断。

## 何时使用
- 一章或一个场景写完之后
- 多个文件可能已经不同步时
- 作者准备把当前版本视为稳定稿之前

## 输入材料
- 当前正文
- 相关设定
- 相关摘要
- 可选的线索、时间线或说话方式说明

## 工作流程
1. 检查人物身份、关系和称呼是否一致
2. 对照摘要检查时间顺序和事件先后
3. 对照线索和世界规则检查是否冲突
4. 报告具体问题及其位置和原因
5. 给出可选修正建议，但不自动覆盖原稿

## 输出结果
- 一份问题清单
- 每个问题对应的简短修正建议

## 约束条件
- 不自动整章重写
- 除非影响连续性，否则不评价文笔质量
- 区分"已确认冲突"和"可能存在风险"

## 可定制项
- 增加专门的线索表或时间线文件
- 增加说话方式、视角稳定性等专项检查
