# 小说项目规则

## 工作原则
- 优先读取已有文档，不要脱离工作区凭空续写
- 尊重作者给定的大纲和设定，不擅自修改主线
- 信息不足时先保守处理，不自造关键设定
- 修改正文时优先局部修改，避免整章推翻重写

## Skill 使用规则
- 头脑风暴类 Skill 负责逼清约束，不负责代替作者决定故事优劣
- 扩写类 Skill 开始前必须先读提纲、设定和相关摘要
- 检查类 Skill 只输出问题清单和建议，不直接覆盖原稿

## 写作顺序
- 开写新章节前，先读取总纲、相关人物设定、上一章摘要、本章提纲
- 写完章节后，补写本章摘要
- 当人物关系、能力、目标发生变化时，同步更新设定文件

## 输出要求
- 正文以小说正文为主，不夹带解释性说明
- 摘要只记录事实变化，不复述修辞
- 需要改 Skill 时，优先在原有结构上增补，不要整份改写成另一套风格
